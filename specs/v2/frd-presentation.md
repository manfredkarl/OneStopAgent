## 1. Overview

The Presentation agent generates a PowerPoint deck from all agent outputs using the `python-pptx` library. It compiles architecture, services, costs, and business value into an 8-slide deck ready for customer presentations. The generated file is saved to the `output/` directory and is downloadable via the API.

---

## 2. Input

The agent reads the full `AgentState`:

| Field | Source | Description |
|-------|--------|-------------|
| `state.user_input` | User | Project description |
| `state.customer_name` | User | Customer name (defaults to "Customer") |
| `state.architecture` | Architect | Mermaid code, components, narrative |
| `state.services` | Azure Specialist | Service selections with SKUs |
| `state.costs` | Cost Specialist | Cost estimate with items and totals |
| `state.business_value` | Business Value | Value drivers and executive summary |

**Context assembly:**

```python
context = {
    "customerName": state.customer_name,
    "description": state.user_input,
    "architecture": state.architecture,
    "serviceSelections": state.services,
    "costEstimate": state.costs,
    "businessValue": {"assessment": state.business_value}
}
```

## 3. Slide Structure

The deck contains 8 slides generated via `create_pptx(context)`:

| # | Slide Title | Content Source | Layout |
|---|-------------|---------------|--------|
| 1 | "Azure Solution Proposal ‚Äì {customer}" | `customer_name` | Title slide (index 0) |
| 2 | "Customer Requirements" | `user_input` (truncated to 2000 chars) | Content slide (index 1) |
| 3 | "Architecture Overview" | `architecture.narrative` | Text slide (index 1) |
| 4 | "Architecture Components" | `architecture.components` (max 12 items) | Content slide (index 1) |
| 5 | "Selected Azure Services" | `services.selections` (max 12 items) | Content slide (index 1) |
| 6 | "Cost Estimate" | `costs.estimate.items` (max 10) + totals | Content slide (index 1) |
| 7 | "Business Value & ROI" | `business_value.drivers` + `executiveSummary` | Content slide (index 1) |
| 8 | "Next Steps" | Static recommendations | Content slide (index 1) |

### 3.1 Slide Details

**Slide 1 ‚Äî Title:**

- Title: `"Azure Solution Proposal ‚Äì {customer}"`
- Subtitle: `"Generated by OneStopAgent"`
- Uses PowerPoint title slide template (layout index 0)

**Slide 2 ‚Äî Requirements:**

- Title: `"Customer Requirements"`
- Body: `description[:2000]` (truncated to prevent overflow)

**Slide 3 ‚Äî Architecture Overview:**

- Title: `"Architecture Overview"`
- Body: `architecture.narrative` (2-3 sentence description)

**Slide 4 ‚Äî Architecture Components:**

- Title: `"Architecture Components"`

  ```
  ‚Ä¢ {name}: {azureService} ‚Äì {description}
  ```

**Slide 5 ‚Äî Azure Services:**

- Title: `"Selected Azure Services"`
- Bullet list (max 12 items):
  ```
  ‚Ä¢ {serviceName} ({sku}) ‚Äì {componentName}
  ```

**Slide 6 ‚Äî Cost Estimate:**

- Title: `"Cost Estimate"`
- Bullet list (max 10 items):
  ```

  ```

**Slide 7 ‚Äî Business Value & ROI:**

- Title: `"Business Value & ROI"`
- Bullet list of value drivers:
  ```
  ‚Ä¢ {name}: {impact}
  ```

**Slide 8 ‚Äî Next Steps:**

## 4. PowerPoint Generation

### 4.1 Library

Uses `python-pptx` for programmatic PowerPoint generation.

### 4.2 Slide Layout Methods

| Method | Purpose | Layout Index |
|--------|---------|-------------|
| `_add_title_slide(prs, title, subtitle)` | Title page with centered text | 0 |
| `_add_content_slide(prs, title, bullets)` | Slide with title and bullet list | 1 |
| `_add_text_slide(prs, title, body_text)` | Slide with title and body paragraph | 1 |

### 4.3 Output Path

```python
out_dir = os.path.join(os.path.dirname(os.path.dirname(__file__)), "output")
# Resolves to: {repo_root}/src/python-api/output/

filename = f"OneStopAgent-{customer.replace(' ', '_')}-{uuid.uuid4().hex[:8]}.pptx"
# Example: OneStopAgent-Acme_Corp-a1b2c3d4.pptx

filepath = os.path.join(out_dir, filename)
```

The `output/` directory is created automatically if it doesn't exist.

## 7. Error Handling

| Failure | Fallback |
|---------|----------|
| Missing architecture data | Skip architecture slides (3, 4) or use placeholders |
| Missing services data | Skip services slide (5) or show "No services selected" |
| Missing cost data | Skip cost slide (6) or show "Cost estimate not available" |
| Missing business value data | Skip BV slide (7) or show "Business value analysis not available" |
| `python-pptx` error | Log error, return error message to pipeline |
| Output directory doesn't exist | Create it automatically (`os.makedirs`) |

## 9. Design Notes

- **Max items per slide:** Components and services capped at 12, costs at 10 ‚Äî prevents text overflow and keeps slides readable
- **Description truncation:** Requirements truncated to 2000 chars, executive summary to 300 chars ‚Äî fits PowerPoint text boxes
- **Static next steps:** Intentionally generic and always applicable ‚Äî no LLM call needed
- **Customer name handling:** Spaces replaced with underscores in filename; defaults to "Customer" if not provided
- **No Mermaid in PPTX:** Mermaid diagrams cannot be rendered in PowerPoint. Slide 3 uses the narrative text instead. Future enhancement could render Mermaid to PNG and embed as an image.
